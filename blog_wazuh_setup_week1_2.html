Wazuh Monitoring Setup in My Homelab: Real-Time Threat Detection Made Simple

How I built an enterprise-grade SIEM environment inside my Proxmox homelab

Published: Week 1, Thursday • Author: Jonathan Kirui • Tags: Wazuh • SIEM • Threat Detection • Homelab • Monitoring

When I first started building my homelab, I knew I needed more than just virtualization — I needed visibility. That’s where Wazuh, a free and open-source SIEM platform, came in. In this post, I’ll walk through how I deployed Wazuh in my homelab and how it’s helped me simulate blue team operations like real-time log monitoring, file integrity checks, and CVE detection.

🔍 Why Wazuh?

Wazuh is the perfect choice for homelab security monitoring because it:
• Supports log collection, file integrity monitoring, and vulnerability detection
• Offers a powerful dashboard via Kibana or its native UI
• Integrates with Linux, Windows, firewalls & cloud services
• Runs efficiently even in single-node setups

🏗️ My Deployment Architecture

• Host: Ubuntu Server LXC container in Proxmox
• Wazuh Version: 4.5
• Stack: Wazuh Manager + Wazuh Dashboard + Elasticsearch
• Agents Installed On:
– Kali Linux
– Windows 10 VM
– Ubuntu Docker Host
– TurnKey File Server
– pfSense Firewall (via syslog forwarding)

I kept it modular using Docker where possible to simplify upgrades and backups.

“A properly configured SIEM gives life to your homelab — it turns noise into intelligence.”

🔐 What It Monitors

In my homelab, Wazuh helps me monitor:
• File Integrity Monitoring – Detect unauthorized changes
• Log Collection – Aggregate system & security logs
• CVE Detection – Identify unpatched vulnerabilities
• Rootkit Detection – Scan for hidden malware tools
• Real-Time Alerts – Email / Slack notifications
• Sysmon Integration – Deep Windows monitoring

One standout success: Wazuh helped me detect CVE-2025-29824 in real time and respond before it became a threat.

⚙️ Agent Deployment Example (Ubuntu)

curl -s https://packages.wazuh.com/4.5/wazuh-install.sh | sudo bash
sudo /var/ossec/bin/agent-auth -m [manager-ip]
sudo systemctl enable wazuh-agent && sudo systemctl start wazuh-agent

📈 Wazuh Dashboard Insights

From the dashboard I can view:
• Threat levels by host
• Attack surface exposure
• Detected CVEs by severity
• Agent health status

This transforms my homelab into a mini SOC (Security Operations Center).

💡 What I’ve Learned

• SIEMs aren’t just for enterprise environments
• Real-time alerting improves detection and response mindset
• Monitoring forces you to improve your own security posture
