Hosting My Developer Portfolio on GitHub Pages – What I Learned

Published: Monday, 16 March 2026 • Author: Mark Andrew Wanjau

When I decided to put my developer portfolio online, I wanted something simple, reliable, and free.

I did not want to worry about servers, billing dashboards, or “trial expiring in 7 days” emails.

I just wanted a place to host my website and focus on building my content.

So I went with GitHub Pages.

And honestly, it turned out to be one of the most straightforward decisions I have made in this journey.

Why GitHub Pages Made Sense

GitHub Pages feels like it understands developers. It is quiet, stable, and does exactly what it needs to do without getting in your way.

Here is what stood out to me:

✔ Completely free
✔ No backend setup required
✔ Deployment is basically “push and it is live”
✔ Built on top of GitHub, where my work already lives

Instead of managing a hosting panel or configuring a VPS, I just pushed my code to a repository and watched the site go live.

That simplicity removed a lot of mental friction.

The Setup Experience

I think a lot of people expect hosting to feel complicated. This one didn’t.

Once I created a repository and enabled GitHub Pages, it simply worked.

The best part is how natural deployments now feel.

I update something, commit, push — and that is it.

No FTP uploads. No deployment pipelines to babysit.

My portfolio updates the same way my projects do. It feels like development should feel — consistent.

What Hosting My Portfolio Taught Me

Putting your work online does something important.

It removes the “private comfort zone” feeling and replaces it with accountability.

When your projects are visible:

• You think more carefully about quality
• You write better descriptions
• You organize things more intentionally

I also realized something simple but meaningful:

A portfolio is not just a website. It is a story.

It shows:

• What you are learning
• How you think
• How serious you are about growing

Even if it feels small now, it represents movement.

And movement matters.

Some Honest Thoughts

There are fancier hosting options out there.

But sometimes simple is exactly what you need.

GitHub Pages gives me:

• Reliability
• Speed
• Zero cost
• Peace of mind

I don’t have to babysit anything. I just build.

What’s Next

My portfolio will keep evolving.

I want it to become more than a static site. More projects. More learning logs. More real things I am working on.

But for now, GitHub Pages does a solid job.

It keeps the focus on the work, not the infrastructure.

If you are building your first developer portfolio, would you host it on GitHub Pages — or choose something like Vercel, Netlify, or a full server setup?

I am curious how other developers think about this choice.
