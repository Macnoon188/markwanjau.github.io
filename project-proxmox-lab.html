Inside My Proxmox Homelab

A Cybersecurity Playground for Real-World Skills and Threat Response

Over the past few months, I’ve been steadily building out a Homelab that not only sharpens my technical skills, but also simulates real-world cybersecurity workflows. Powered by Proxmox, my virtualized lab mimics an enterprise IT environment, complete with Windows Server, pfSense firewall, and a Wazuh SIEM for centralized monitoring.

While it's been a rewarding project from a technical perspective, it’s also become a space where I actively respond to real security threats—one of which I’ll talk about shortly.

🚀 What My Homelab Looks Like

The heart of the lab is Proxmox VE, an open-source virtualization platform that allows me to spin up and manage multiple VMs. Here’s a breakdown of the core setup:

  • Windows Server 2022: Configured to handle DNS, DHCP, and Active Directory roles.
  • Windows 10 Workstation: A domain-joined client VM used to test policy enforcement, user access control, and endpoint behavior.
  • pfSense Firewall: Manages internal segmentation, NAT rules, and acts as a front line for inbound/outbound traffic control.
  • Wazuh 4.5 SIEM: Integrated for real-time log collection, rule-based alerting, and CVE detection.

My goal was never just to "build servers" — I wanted a lab that monitors itself, exposes weaknesses, and allows me to fix them before an attacker would.

🧠 Real Threat, Real Response: CVE-2025-29824

A few weeks ago, my Wazuh instance flagged a suspicious log entry related to a known vulnerability: CVE-2025-29824 — a privilege escalation flaw in the Windows Common Log File System (CLFS) driver. Left unpatched, it allows local users to gain elevated privileges and execute code.

Here’s how I handled it:

  1. Detection – Wazuh alerted me to the vulnerability during a routine scan.
  2. Research – I cross-referenced the CVE on MITRE and learned that Microsoft hadn't released a patch yet.
  3. Mitigation – I found a PowerShell mitigation script via the community at Vicarius (vsociety.io).
  4. Execution – Deployed the script across the affected systems in my lab.
  5. Validation – Confirmed the hardening both manually and via follow-up Wazuh logs.

✅ Key Takeaways:

  • SIEM visibility is crucial even in test environments.
  • Community tools can bridge the gap when official patches aren’t yet available.
  • Your Homelab isn’t just a project—it’s a battlefield to practice incident response and threat hunting.

🛠️ Skills I’ve Built Along the Way

  • Virtualization & system administration (Proxmox, Windows Server roles)
  • Network security (pfSense configuration, firewall rules, VLANs)
  • SIEM management & alert triage (Wazuh 4.5)
  • Traffic analysis with Wireshark and tcpdump
  • Ethical hacking tools via Kali Linux (planned)
  • Incident detection and response for CVEs

🎯 Why This Matters

In cybersecurity, theory isn't enough. You need a place to practice detection, response, and mitigation. My Proxmox Homelab has become just that:

  • A controlled sandbox for experimentation
  • A space to simulate enterprise-level scenarios
  • A launchpad to hone my investigative mindset

And the best part? Every alert, misconfiguration, or CVE is a new learning opportunity.

📎 Final Thoughts

Building and maintaining a Homelab isn't just a hobby—it's an investment in becoming a better cybersecurity professional. Whether it's catching live CVEs or refining my approach to segmentation and access control, each piece of the puzzle adds to my understanding and readiness for the real world.
