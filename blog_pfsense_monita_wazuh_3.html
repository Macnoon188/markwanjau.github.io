Monitoring pfSense with Wazuh: A Step-by-Step Guide for My Homelab

How I integrated my firewall with Wazuh SIEM to gain real-time network visibility

Published: Week 2, Tuesday • Author: Mark Andrew Wanjau • Tags: Wazuh • pfSense • SIEM • Cybersecurity • Monitoring

When I first integrated pfSense into my homelab, I realized that while it’s a powerful firewall, it needed centralized monitoring to fully understand what was happening at the network perimeter. Since I already run Wazuh 4.5 for endpoint monitoring, connecting pfSense logs to my SIEM was the logical next step. This guide explains how I did it and the benefits I gained.

🔒 Why Monitor pfSense with Wazuh?

pfSense is the gatekeeper of your homelab. Feeding its logs into Wazuh gives:
• Real-time firewall event visibility
• Insight into blocked connections, scans & VPN events
• Correlation between firewall + endpoint alerts
• A single SIEM dashboard for your entire lab

This turns a basic homelab into a lightweight SOC training environment — perfect for blue-team learning.

🏗️ My Homelab Setup

• Firewall: pfSense CE (dedicated hardware)
• SIEM: Wazuh 4.5 on Ubuntu (Proxmox LXC)
• Log Transport: Syslog (UDP/TCP)
• Wazuh Module: Logcollector

Before starting, ensure Wazuh Manager can reach pfSense over LAN.

🔧 Step-by-Step Integration

Step 1 — Enable Remote Logging

pfSense → Status → System Logs → Settings → Remote Logging Options
Enable & add your Wazuh Manager:
192.168.x.x:514
Select logs:
• System • Firewall • DHCP • VPN • DNS Resolver

Step 2 — Configure Wazuh

File: /var/ossec/etc/ossec.conf

<remote>
  <connection>syslog</connection>
  <port>514</port>
  <protocol>udp</protocol>
  <allowed-ips>192.168.0.0/24</allowed-ips>
</remote>

Restart Wazuh:

sudo systemctl restart wazuh-manager

Step 3 — Optional: Apply Parsing Rules

File: /var/ossec/etc/rules/pfsense_rules.xml

<group name="pfsense,">
  <rule id="700001" level="5">
    <decoded_as>syslog</decoded_as>
    <match>filterlog:</match>
    <description>pfSense Firewall Event</description>
  </rule>
</group>

Restart again:

sudo systemctl restart wazuh-manager

📊 Verifying Logs

In Wazuh Dashboard → Security Events, filter by:
• pfSense Source IP
• Rule group “pfsense”

You should see:
• Blocked inbound traffic
• DHCP leases
• VPN events
• DNS resolver logs
• Port scan detections

🔍 What I Monitor

• Firewall events → intrusion attempts
• DHCP → new / rogue devices
• VPN logs → access monitoring
• DNS → suspicious domains
• System logs → reboots & interface issues

🎯 Key Benefits Gained

• Better correlation between network & host threats
• Full visibility into WAN attacks
• Real-world SIEM + firewall experience
• Stronger lab security posture
