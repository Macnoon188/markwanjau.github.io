Monitoring pfSense with Wazuh

Turning My Firewall Into a Real Visibility Layer

Published: January 2026 • Author: Mark Andrew Wanjau • Tags: Wazuh • pfSense • SIEM • Monitoring

When I rebuilt my homelab, I stopped treating cybersecurity like a tools experiment and started treating it like a real network.

pfSense became the backbone of my environment. But having a firewall is one thing. Seeing what it is doing is another.

That is where Wazuh came in.

Step 1: Configure pfSense Remote Logging

Navigate to:

Status → System Logs → Settings

Enable Remote Logging and add your Wazuh Manager IP:


192.168.x.x:514

Select logs such as:

• System
• Firewall
• DHCP
• VPN
• DNS Resolver

Step 2: Configure Wazuh to Receive Logs


File: /var/ossec/etc/ossec.conf


  syslog
  514
  udp
  192.168.0.0/24

Restart Wazuh:


sudo systemctl restart wazuh-manager

Optional: Add Custom Parsing Rules


File: /var/ossec/etc/rules/pfsense_rules.xml


  
    syslog
    filterlog:
    pfSense Firewall Event

Restart the manager again after adding rules:


sudo systemctl restart wazuh-manager

What I Started Seeing

• Blocked inbound traffic
• Port scans
• Failed connection attempts
• DHCP activity
• VPN events

What This Taught Me

Security is not about having tools.
It is about understanding what they are telling you.

pfSense generates the story. Wazuh helps interpret it.
