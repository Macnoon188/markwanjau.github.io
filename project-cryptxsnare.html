Why I Built This

Most people learn cybersecurity by studying tools. I wanted to understand how those tools actually work — so I built one.

Not a mock dashboard. Not a UI concept. A full SIEM-style platform that simulates how a Security Operations Centre actually functions.

My path into cybersecurity didn't begin in a lab. It started at the helpdesk. Password resets. Printer issues. Outlook errors. The usual IT support cycle. But over time I started noticing patterns that bothered me — systems were misconfigured, users clicked obvious phishing links, and there was almost no visibility into what was actually happening on the network.

One question kept sitting in the back of my mind:

"Who's actually watching for real threats?"

That question pulled me into cybersecurity. And eventually, it led to CryptXSnare.

What CryptXSnare Is

CryptXSnare is a cybersecurity command centre. When you log in, you land in an environment that mirrors what a SOC analyst deals with every day — alerts arrive, cases open, investigations start, analysts work through suspicious activity.

The platform runs in demo mode by default. No credentials, no setup, no friction. You're immediately logged in as an analyst working through simulated threat data. I wanted people to experience the workflow, not just read about it.

Core Features

📊 SIEM Dashboard

Real-time KPIs, alert trends, and severity distribution for quick threat landscape visibility.

🔔 Alert Management

Full alert lifecycle — triage, acknowledgement, assignment, and resolution tracking.

📁 Case Management

Group related alerts into investigations with status tracking and analyst ownership.

🌐 URL & IP Analysis

Threat scoring using intelligence sources including VirusTotal and AbuseIPDB.

📧 Email Header Analysis

Parses SPF, DKIM, and DMARC records to help identify phishing attempts.

📡 Network Monitoring

Traffic pattern analysis and suspicious flow detection for anomaly simulation.

🕵️ OSINT Reconnaissance

Passive intelligence — WHOIS, DNS records, subdomains, and technology fingerprinting.

🧠 Threat Intelligence

IOC tracking with confidence scores and source attribution.

📂 File Analysis

Hash reputation checks against known malicious file databases.

⚡ SOAR Automation

Playbooks that simulate automated responses — block TOR nodes, quarantine files, isolate hosts.

📄 Reporting Engine

Generate structured investigation reports. If it isn't documented, it didn't happen.

📈 Analytics

Trend analysis across alerts and cases for communicating findings to stakeholders.

The Moment Scope Nearly Killed It

Halfway through building, I ran into a problem most developers know well: feature creep. Every module sparked a new idea. "It would be cool if this also did…"

That thinking almost derailed the project. I had two choices — keep expanding forever, or build something structured and finish it. I chose structure.

Instead of cramming features together, I designed a modular architecture with service layers for alerts, cases, recon, and threat intelligence. That decision made the project manageable and future-proof.

Run It Yourself

git clone github.com/deekim254/cryptXsnare npm install npm run dev

Then open http://localhost:5173 — you'll be logged in automatically as an analyst in demo mode.

To enable full backend functionality, connect Supabase and deploy the included Edge Functions.
